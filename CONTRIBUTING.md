# Contributing to Ultrathink Custom-Plugin-MLOps

Welcome! We're excited to have you contribute to Ultrathink's comprehensive developer learning platform.

## How to Contribute

### 1. **Improve Agent Documentation**
Agents are the core of our plugin. Help make them more comprehensive:
- Add new roles within existing agents
- Expand learning timelines with more detailed milestones
- Include more real-world examples from your experience
- Add salary data, job market insights

### 2. **Enhance SKILL.md Files**
Skills need continuous improvement:
- Add advanced topics and production patterns
- Document common gotchas and pitfalls
- Add best practices from your experience
- Include links to relevant resources
- Add code examples and tutorials

### 3. **Expand Project Library**
Help developers practice:
- Add new hands-on projects
- Include project solutions and walkthroughs
- Add difficulty assessments
- Include learning outcomes for each project
- Add estimated project duration

### 4. **Improve Commands**
Make commands more interactive:
- Add workflow examples
- Include case studies
- Add more detailed guidance
- Improve command documentation

### 5. **Add Missing Specializations**
Identify and add new roles:
- Game Development patterns
- AR/VR development
- Embedded systems
- Quantum computing (emerging)
- New AI/ML specializations

## Contribution Guidelines

### Code Style
- Use consistent formatting (follow existing patterns)
- Include YAML frontmatter in agent and skill files
- Use clear, professional language
- Add real-world examples

### Content Standards
- **Beginner content**: Clear, well-explained, with examples
- **Intermediate content**: Assume some knowledge, focus on depth
- **Advanced content**: Focus on patterns, optimization, architecture
- **Always include**: Why this matters, real-world applications

### Documentation Quality
- Add references to official documentation
- Include links to high-quality resources
- Cite real companies and case studies
- Include estimated learning times

### Testing Contributions
1. **Verify completeness**: No missing sections
2. **Check consistency**: Matches other agents/skills
3. **Test commands**: Make sure examples actually work
4. **Validate links**: All resources are current

## Development Setup

```bash
# Clone the repository
git clone https://github.com/pluginagentmarketplace/custom-plugin-mlops.git
cd custom-plugin-mlops

# Create your feature branch
git checkout -b feature/your-feature-name

# Make your changes to:
# - agents/*.md
# - skills/**/SKILL.md
# - commands/*.md

# Test your changes locally in Claude Code
claude code load ./

# Commit with clear messages
git commit -m "Add/improve: specific contribution description"

# Push to your fork
git push origin feature/your-feature-name

# Create a Pull Request
```

## PR Checklist

Before submitting a PR:
- [ ] Changes follow existing style and format
- [ ] YAML frontmatter is correct (agents and skills)
- [ ] All links are valid and current
- [ ] No breaking changes to existing content
- [ ] Content is technically accurate
- [ ] Added relevant examples or resources
- [ ] Tested locally in Claude Code

## Types of Contributions

### Major Contributions (0.5-2 hours)
- Add new agent or major agent expansion
- Create new comprehensive SKILL.md
- Add 5+ new projects
- Major command improvements

### Medium Contributions (20-60 minutes)
- Enhance existing agent with new roles
- Expand SKILL.md with advanced topics
- Add 2-3 new projects
- Improve command documentation

### Minor Contributions (10-20 minutes)
- Fix typos and grammar
- Update links and resources
- Add small examples
- Clarify documentation
